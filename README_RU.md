# Frontier Industry Calculator

Это локальный браузерный калькулятор производственных цепочек.
Он работает без установки: достаточно открыть страницу из папки `WEB` и загрузить файлы данных через встроенные кнопки.

## Что должно быть в архиве

Обычно архив содержит:

- `WEB/` - папка с самим интерфейсом
- `calculator_graph.json` - готовый граф данных для калькулятора
- `item_icons.zip` - архив с иконками предметов

Если у вас есть именно такой набор, ничего дополнительно собирать не нужно.

## Как запустить

1. Распакуйте архив в любую папку на компьютере.
2. Откройте папку `WEB`.
3. Запустите файл `index.html` в браузере.
4. Нажмите кнопку `Load Data`.
5. В окне загрузки выберите:
   - `Graph JSON` - файл `calculator_graph.json`
   - `Icons ZIP` - файл `item_icons.zip`

После загрузки данных калькулятор начнет работать: можно искать предметы, выбирать цель, задавать количество и смотреть цепочку производства.

## Если папка называется `web`, а не `WEB`

Это не важно. Открывайте ту папку, которая лежит в архиве, и запускайте находящийся внутри `index.html`.

## Что делает каждый файл

- `calculator_graph.json` - основной файл с предметами, рецептами и связями между ними
- `item_icons.zip` - необязательный файл с картинками предметов

Если `item_icons.zip` не загружать, калькулятор все равно будет работать, но вместо иконок могут показываться буквенные заглушки.

## Что есть в интерфейсе

### Верхняя панель

Вверху показываются:

- `Target` - текущая выбранная цель
- `Progress` - прогресс выполнения плана
- `ETA` - ориентировочное время завершения

Справа находятся основные кнопки:

- `Load Data` - загрузка `calculator_graph.json` и `item_icons.zip`
- `Dataset` - сведения о загруженном наборе данных
- `Filters` - фильтрация каталога предметов
- `Default Recipe Paths` - стандартные пути рецептов для ключевых цепочек
- `View` - настройки отображения дерева и списка

Также есть переключатель режимов:

- `Calculator` - обычный режим расчета цепочки производства
- `Planner` - режим планировщика, где можно собирать несколько целевых строк в очередь

### Левая колонка: Target

Здесь выбирается цель расчета.

- `Search` - поиск предмета по названию или `typeID`
- `Results` - список найденных предметов
- `Selected target` - выбранный предмет
- `Quantity` - нужное количество результата
- `Recipe` - выбор альтернативного рецепта, если у предмета их несколько

### Основная область: рабочие вкладки

В центре есть три вкладки:

- `Plan` - основной расчетный экран
- `Pipeline` - группировка производственной цепочки по стадиям
- `Tree` - древовидный вид всей зависимости

#### `Plan`

Вкладка показывает основные рабочие блоки:

- `Next Actions` - ближайшие действия, которые нужно выполнить в первую очередь
- `Bottleneck` - текущий узкий участок цепочки
- `Materials` - сырье, которое нужно добыть
- `Components` - компоненты, которые нужно произвести

### Правая/вспомогательная часть

Через дополнительные панели можно открыть:

- `Dataset` - краткая сводка по загруженному графу и текущему плану
- `Filters` - каталог доступных крафтовых предметов и ограничение области поиска
- `Default Recipe Paths` - базовые рецептурные пути для отдельных корневых предметов
- `View` - управление видом дерева: раскрыть все, свернуть все и переключать отображение деталей

### Режим `Planner`

В режиме `Planner` интерфейс меняется на конструктор очереди:

- слева можно искать крафтовые предметы и добавлять их в план
- в центре показывается сводка производства
- справа отображаются сырье, компоненты и общая сводка по выбранным строкам

Внутри этого режима можно:

- добавлять и удалять строки плана
- менять количество для каждой строки
- отмечать прогресс по материалам и компонентам
- переключаться между несколькими целями в одной производственной сессии

## Если данных нет или загрузка не работает

- Убедитесь, что `calculator_graph.json` действительно лежит в архиве и выбран в поле `Graph JSON`
- Убедитесь, что `item_icons.zip` не распакован вручную и загружается именно как ZIP-архив

## Коротко

Если у вас есть архив с `WEB`, `calculator_graph.json` и `item_icons.zip`, то схема такая:

1. Распаковать архив.
2. Открыть `WEB/index.html`.
3. Загрузить `calculator_graph.json`.
4. Загрузить `item_icons.zip`.
5. Пользоваться калькулятором прямо в браузере.
